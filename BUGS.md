### BUG-UI-001 — Неверный термин «Щенок» в фильтре «Возраст» для категории Кошки

| Поле | Значение |
|------|----|
| **Severity** | P1 |
| **Компонент** | Фильтры боковой панели |

**Описание:**

**Ожидаемый результат:** В фильтре «Возраст» отображается «Котёнок».
**Фактический результат:** В фильтре «Возраст» отображается «Щенок».

### BUG-UI-007 — Фильтр «Диапазон цен» прибавляет 5000 к введённому значению

| Поле | Значение |
|------|----|
| **Severity** | P1 |
| **Компонент** | Фильтры / Диапазон цен |
| **Среда** | https://cerulean-praline-8e5aa6.netlify.app |

**Описание:**
В компоненте фильтра цены к значению из полей «От» и «До» перед отправкой в API прибавляется 5 000. Код обоих обработчиков идентичен:
```js
// поле «От» → minPrice
const h = value === "" ? undefined : parseFloat(value);
dispatch(setMinPrice(h === undefined ? undefined : h + 5000));

// поле «До» → maxPrice
const h = value === "" ? undefined : parseFloat(value);
dispatch(setMaxPrice(h === undefined ? undefined : h + 5000));
```
Если пользователь вводит `1000` в поле «До», API получает `maxPrice=6000`.

**Шаги воспроизведения:**
1. Открыть список объявлений.
2. Ввести в поле «До» значение `1000`.
3. Проверить сетевые запросы (DevTools → Network).

**Ожидаемый результат:** Запрос уходит с `maxPrice=1000`.
**Фактический результат:** Запрос уходит с `maxPrice=6000`.

### BUG-UI-009 — Кнопка «Включить автообновление» на странице статистики не работает

| Поле | Значение |
|------|----|
| **Severity** | P1 |
| **Компонент** | Таймер автообновления / Страница статистики |
| **Среда** | https://cerulean-praline-8e5aa6.netlify.app/stats |

**Описание:**
После остановки автообновления кнопка «▶️» не реагирует на нажатие. Код обработчика:
```js
const a = () => { n && (o(false), s(t / 1000)) };
```
Условие `n &&` означает, что функция выполняется только когда `n === true` (автообновление активно). Когда автообновление выключено (`n === false`), клик игнорируется.

**Шаги воспроизведения:**
1. Открыть `/stats`.
2. Нажать кнопку «⏸» (Отключить автообновление).
3. Нажать появившуюся кнопку «▶️» (Включить автообновление).

**Ожидаемый результат:** Таймер запустился, появился обратный отсчёт «Обновление через: X:XX».
**Фактический результат:** Ничего не происходит. Таймер не запускается.

### BUG-UI-020 — Страница статистики отображает случайные некорректные данные

| Поле | Значение |
|------|----|
| **Severity** | P1 |
| **Компонент** | Страница статистики `/stats` |
| **Среда** | https://cerulean-praline-8e5aa6.netlify.app/stats |

**Описание:**
Показатели на странице `/stats` генерируются случайно при каждой загрузке и содержат математически некорректные значения:
- **«Одобрено» + «Отклонено» ≠ 100%**: наблюдалось 23% + 64% = **87%** — пропадают 13% объявлений.
- **Среднее время проверки**: показывалось **11 139 минут** (~185 часов) на одно объявление — заведомо нереалистичное значение.
- Данные меняются при каждом обновлении страницы.

**Шаги воспроизведения:**
1. Открыть `/stats`.
2. Запомнить «Одобрено» и «Отклонено» в процентах.
3. Обновить страницу — значения изменятся.
4. Сложить «Одобрено» и «Отклонено».

**Ожидаемый результат:** «Одобрено» + «Отклонено» = 100%; среднее время проверки — реалистичное число.
**Фактический результат:** Сумма процентов ≠ 100%, значения случайные при каждой загрузке.
